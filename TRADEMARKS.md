## Permitted Use

You may use the Terajs name to:

- reference the framework in tutorials, articles, videos, or documentation  
- describe that your project *uses* Terajs  
- accurately identify Terajs as a dependency  
- build libraries, tools, or integrations that work with Terajs  

These uses must be **truthful**, **non‑misleading**, and **non‑confusing**.

---

## Restricted Use

You **may not**:

- use the name **Terajs** in your own framework, library, or tool in a way that implies official affiliation  
- use the Terajs name or logo in your project title  
- publish a fork called “Terajs”, “TerajsJS”, “Terajs Framework”, or anything confusingly similar  
- use the Terajs name in domain names, npm package names, GitHub org names, or branding  
- imply endorsement or partnership without written permission  

Forks must use **clearly different names**.

## Logo Usage

The Terajs logo (when published) may not be used without permission, except for:

- news, commentary, or educational use  
- documentation that references Terajs factually

## Why This Exists

Open‑source code is Apache‑licensed.  
The **brand** is trademark‑protected to:

- prevent impersonation  
- avoid ecosystem confusion  
- protect users  
- ensure long‑term project identity
