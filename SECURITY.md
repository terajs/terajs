# Security Policy

## Supported Versions

Security fixes are prioritized for the latest release line published from this repository.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

If private vulnerability reporting is enabled for this repository, use GitHub's private reporting flow. Otherwise, contact the maintainers privately before sharing details publicly.

When you report an issue, include:

- the affected package, workflow, or deployment path
- reproduction steps or a proof of concept
- the impacted version or commit range
- any suggested mitigation if you already have one

We will triage reports, confirm impact, and coordinate a fix and disclosure window before public release when possible.
